# RAG Pipeline Manager — 사용 설명서

**앱 버전**: v1.32  
**실행 환경**: Windows 11 · Python 3.12 · Streamlit  

---

## 1. 앱 시작

```bash
streamlit run app.py
```

브라우저에서 `http://localhost:8501` 로 자동 열립니다.  
처음 실행 시 모델 로드(EasyOCR GPU 등)로 수십 초가 소요될 수 있습니다.

---

## 2. 화면 구성 개요

```
┌─────────────────────────────────────────────────────┐
│  사이드바 (좌)         메인 영역 (우)                  │
│  ─ 파이프라인 실행     ─ 요약 지표 (5개 메트릭)         │
│  ─ 새로고침            ─ 탭 6개                        │
│  ─ 전체 재OCR          │                               │
└─────────────────────────────────────────────────────┘
```

## 3. 사이드바

### 3-1. 파이프라인 실행

| 버튼 | 기능 |
|------|------|
| **▶ 파이프라인 실행** | `data/pdf_watch` 폴더를 스캔하여 NEW 문서를 탐지·처리·AnythingLLM 등록까지 일괄 실행 |

- 실행 중에는 버튼이 비활성화되고 "파이프라인 실행 중..." 메시지 표시
- 완료 후 아래 코드 블록에 결과 요약 표시  
  예: `신규 탐지: 3건 / 처리 완료: 3건 / INDEXED: 2건 / FAILED: 1건`

### 3-2. 새로고침

| 버튼 | 기능 |
|------|------|
| **🔄 새로고침** | DB 캐시를 초기화하고 문서 목록을 즉시 다시 로드 |

화면 아래에 마지막 갱신 시각이 표시됩니다 (자동 TTL: 30초).

### 3-3. 전체 재OCR

| 버튼 | 기능 |
|------|------|
| **🔁 전체 재OCR** | DB의 모든 문서를 `NEW` 상태로 초기화 |

> 초기화 후 반드시 **▶ 파이프라인 실행**을 눌러야 실제 재처리가 시작됩니다.  
> 전체 문서를 처음부터 다시 처리하므로 시간이 오래 걸릴 수 있습니다.

## 4. 요약 지표 (상단 메트릭)

화면 상단에 5개의 숫자 카드가 표시됩니다.

| 지표 | 설명 |
|------|------|
| **전체 문서** | DB에 등록된 전체 파일 수 |
| **🎯 INDEXED** | AnythingLLM에 성공적으로 등록 완료된 문서 수 |
| **⚠️ 검토 필요** | OCR 품질이 기준에 미달하거나 수동 확인이 필요한 문서 수 |
| **❌ 실패** | 처리 중 오류가 발생한 문서 수 |
| **🆕 대기 중** | 아직 처리가 시작되지 않은 신규 문서 수 |

## 5. 처리 상태 코드

각 문서는 아래 상태 중 하나를 가집니다.

| 이모지 | 상태 | 의미 |
|--------|------|------|
| 🆕 | NEW | 탐지됨, 처리 대기 중 |
| 🔍 | ANALYZING | PDF 분석 및 전략 선택 중 |
| 📝 | TEXT_EXTRACTED | 디지털 텍스트 추출 완료 |
| 📝 | OCR_DONE | OCR 처리 완료 |
| ✅ | TEXT_READY | 텍스트 정제 완료, 업로드 대기 |
| ⬆️ | UPLOAD_REQUESTED | AnythingLLM 업로드 요청됨 |
| ☁️ | UPLOADED | 업로드 완료, 파싱/임베딩 대기 |
| ⚙️ | PARSING_DONE | 문서 파싱 완료 |
| 🔧 | CHUNKING_DONE | 청킹 완료 |
| 🧩 | EMBEDDING_DONE | 임베딩 완료 |
| 🎯 | INDEXED | 색인 완료 (최종 성공) |
| ⚠️ | REVIEW_REQUIRED | 검토 필요 (품질 미달) |
| ❌ | FAILED | 처리 실패 |

## 6. 탭별 기능

### 탭 1 — 📋 전체 파일 목록

등록된 모든 문서를 보여줍니다.

#### 검색 및 필터
- **파일명 검색**: 파일명의 일부를 입력하면 실시간으로 필터링됩니다.
- **상태 필터**: 특정 상태(INDEXED, FAILED 등)만 선택하여 볼 수 있습니다.

#### 문서 행 (Expander)
각 문서를 클릭하면 상세 정보가 펼쳐집니다.

**정보 영역 (좌):**

| 항목 | 설명 |
|------|------|
| ID / 상태 | 문서 고유 번호와 현재 처리 상태 (색상 배지) |
| 경로 | 파일 전체 경로 |
| 소스 | DIGITAL(텍스트 레이어 있음) / SCANNED(이미지 기반) |
| 레이아웃 | SINGLE_COLUMN / MULTI_COLUMN |
| 언어 | 감지된 언어 (kor, eng 등) |
| 수식 | 수식 포함 여부 |
| OCR 전략 | 적용된 OCR 방식 (아래 표 참조) |
| OCR 품질 점수 | 0.0 ~ 1.0 (높을수록 인식 품질 양호) |
| 오류 메시지 | 실패·검토 사유 |
| 등록/갱신 시각 | 문서가 처음 등록된 시각과 마지막으로 상태가 변경된 시각 |
| 재시도 횟수 | 자동 재처리가 시도된 횟수 |

**OCR 전략 목록:**

| 전략 | 적용 조건 |
|------|----------|
| DIGITAL_EXTRACT | 디지털 PDF, 단일 컬럼 |
| DIGITAL_EXTRACT_MULTI | 디지털 PDF, 다단 컬럼 |
| OCR_KOR_ENG | 스캔 이미지, 한국어+영어 (EasyOCR GPU) |
| OCR_ENG | 스캔 이미지, 영어 전용 (Tesseract) |
| OCR_MULTI_COL | 스캔 이미지, 다단 컬럼 (Tesseract) |
| OCR_FORMULA | 수식 포함 문서 (Tesseract) |
| DOCX_EXTRACT | Word 문서 (.docx) |
| PPTX_EXTRACT | 프레젠테이션 (.pptx) |

**처리 로그 (Expander 안의 Expander):**
- 각 처리 단계(ANALYZE, OCR, UPLOAD 등)의 성공/실패 이력과 타임스탬프를 확인할 수 있습니다.

**액션 영역 (우):**

| 버튼 | 표시 조건 | 기능 |
|------|----------|------|
| 📖 원본/OCR 비교 (체크박스) | 항상 | PDF 뷰어 열기/닫기 |
| 🔄 즉시 재처리 | REVIEW_REQUIRED / FAILED | 재시도 카운터 초기화 후 즉시 재처리 |
| ➡️ NEW 초기화 | REVIEW_REQUIRED / FAILED | 상태를 NEW로만 초기화 (처리는 나중에) |
| 🔁 강제 재처리 | INDEXED | 이미 성공한 문서를 강제로 다시 처리 |

---

### 탭 2 — ⚠️ 검토 필요

`REVIEW_REQUIRED` 상태인 문서만 모아서 보여줍니다.

- **표시 정보**: 파일명, 검토 사유, OCR 전략, 품질 점수, 처리 로그
- **액션**: 🔄 즉시 재처리 / ➡️ NEW 초기화

> 검토 사유 예시:  
> - "OCR 품질 점수(0.32)가 임계값(0.50) 미만"  
> - "추출된 텍스트 길이가 너무 짧음"

---

### 탭 3 — ❌ 실패 목록

`FAILED` 상태인 문서만 모아서 보여줍니다.

- **표시 정보**: 파일명, 실패 단계, 재시도 횟수, 오류 메시지 (빨간 박스), 처리 로그
- **액션**: 🔄 즉시 재처리 / ➡️ NEW 초기화

---

### 탭 4 — 📊 대시보드

전체 문서의 통계를 한눈에 보여줍니다.

#### 상태별 분포 테이블
- 상태 이름, 건수, 전체 대비 비율(%)

#### PDF 분류 통계
- **소스 유형별** 건수: DIGITAL / SCANNED
- **레이아웃 유형별** 건수: SINGLE_COLUMN / MULTI_COLUMN
- **수식 포함** 건수

#### OCR 전략 분포
- 전략별 건수와 막대 그래프(텍스트 기반)

#### OCR 품질 점수 통계
- 평균, 최솟값, 최댓값
- 구간별 분포:
  - `0.0 ~ 0.5` 낮음
  - `0.5 ~ 0.7` 보통
  - `0.7 ~ 0.85` 양호
  - `0.85 ~ 1.0` 우수

---

### 탭 5 — 🖼️ 표·그림 자산

파이프라인이 추출한 표(TABLE)와 그림(FIGURE) 이미지를 확인합니다.

#### 사용 방법
1. **문서 선택** 드롭다운에서 확인할 문서를 선택합니다.  
   - 각 항목에 `TABLE: N건 / FIGURE: N건` 건수가 표시됩니다.
2. **유형 필터** 라디오 버튼으로 전체 / TABLE만 / FIGURE만 선택합니다.

#### 자산 상세 (Expander)
각 자산을 클릭하면 좌우 분할 뷰가 펼쳐집니다.

| 좌측 | 우측 |
|------|------|
| 추출된 이미지 (캡션 포함) | 참조 태그, 페이지 번호, 유형, 캡션, 위치 좌표(pt), OCR 텍스트 |

---

### 탭 6 — 🔬 PDF 분석 진단

특정 PDF의 레이아웃 구조(단일/다단 컬럼)를 수동으로 진단합니다.

#### 사용 방법
1. **PDF 파일 경로** 입력 필드에 분석할 파일 경로를 입력합니다.  
   예: `data/pdf_watch/paper.pdf`
2. 파일이 존재하면 자동으로 분석을 시작합니다 (최대 5페이지).

#### 진단 결과

각 페이지별로 Expander가 생성되며 다음 정보를 보여줍니다.

| 항목 | 설명 |
|------|------|
| 감지 결과 | 🟢 MULTI_COLUMN / 🔵 SINGLE_COLUMN |
| 줄 커버리지 차트 | 페이지 가로 방향(0~100%) 텍스트 밀도 프로파일 |
| 텍스트 줄 수 | 페이지에서 감지된 텍스트 줄 수 |
| 단어 수 | 감지된 총 단어 수 |
| side_mean | 좌우 여백 구간의 평균 커버리지 |
| max_gap | 중앙 구간에서 감지된 최대 공백 폭 (bin 단위) |

> max_gap ≥ 3 이면 다단 컬럼으로 판정합니다.  
> 커버리지 차트에서 중앙부에 뚜렷한 골(골짜기)이 보이면 2단 레이아웃입니다.

## 7. PDF 뷰어 (원본/OCR 비교)

전체 파일 목록에서 **📖 원본/OCR 비교** 체크박스를 체크하면 문서 아래에 뷰어가 열립니다.

### 네비게이션 바

```
[⏮] [◀]  [현재 페이지 입력]  / 전체 페이지  전략: OCR전략  [▶] [⏭]
```

| 버튼 | 기능 |
|------|------|
| ⏮ | 첫 페이지로 이동 |
| ◀ | 이전 페이지 |
| 숫자 입력 | 직접 페이지 번호 입력 후 Enter |
| ▶ | 다음 페이지 |
| ⏭ | 마지막 페이지로 이동 |

### 분할 뷰

| 좌측 | 우측 |
|------|------|
| 원본 PDF 페이지 이미지 | OCR/추출 텍스트 (편집 가능 텍스트 영역) |

- **Word(.docx)**, **PowerPoint(.pptx)** 파일은 원본 미리보기가 지원되지 않으며 우측 텍스트만 표시됩니다.
- 텍스트는 파이프라인이 저장한 DB 결과를 우선 표시하고, DB에 없으면 실시간으로 추출합니다.

## 8. 주요 워크플로우

### 신규 PDF 처리

```
1. data/pdf_watch/ 폴더에 PDF 파일 복사
2. 사이드바 → ▶ 파이프라인 실행 클릭
3. 탭 1 에서 상태 확인 (NEW → ANALYZING → ... → INDEXED)
4. INDEXED 완료 시 AnythingLLM에서 검색 가능
```

### 실패 문서 재처리

```
1. 탭 3 (실패 목록) 또는 탭 1 에서 실패 문서 확인
2. 오류 메시지 / 처리 로그로 원인 파악
3. 🔄 즉시 재처리 → 결과 확인
4. 반복 실패 시 ➡️ NEW 초기화 후 파이프라인 재실행
```

### 전체 재처리 (OCR 설정 변경 후)

```
1. config.yaml 또는 .env 수정 (OCR 전략, LLM 교정 설정 등)
2. 사이드바 → 🔁 전체 재OCR 클릭 → 전체 문서 NEW 초기화
3. 사이드바 → ▶ 파이프라인 실행 클릭
```

## 9. 설정 파일 위치

| 파일 | 역할 |
|------|------|
| `config.yaml` | OCR 옵션, LLM 교정 설정, 감시 디렉토리 등 |
| `.env` | API 키 (AnythingLLM, Anthropic, Google) |
| `data/pdf_watch/` | 자동 감시 폴더 — PDF를 여기에 넣으면 파이프라인이 자동 탐지 |
| `data/rag_project.db` | SQLite DB (문서 상태, 로그, 자산 이미지) |
| `logs/` | 실행 로그 파일 (`log_yyyymmdd_hhmmss.log`) |
